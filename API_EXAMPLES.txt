# RAG CSV Expert - API Examples

This file contains sample API requests and responses for the RAG CSV Expert application.

================================================================================
## 1. UPLOAD FILE
================================================================================

### Request
POST http://127.0.0.1:8000/api/upload
Content-Type: multipart/form-data

file: sales_2023.csv

The data spans from January to December 2023 with daily transaction entries across multiple product categories including Electronics, Clothing, and Home Goods.",

    "row_count: 3245"
  ],
  "metadata": {
    "execution_time": 4.23,
    "data_type": "csv",

  }
}

================================================================================
NOTES:
================================================================================

1. **File Path Optional**: The `file_path` parameter is optional. If omitted, the FileSelectorAgent will automatically determine the most relevant file based on query intent.

2. **Execution Time**: Response times vary based on:
   - LLM provider (Ollama is typically faster for local, Bedrock/Anthropic for cloud)
   - Query complexity
   - Dataset size
   - Whether data is cached in the registry

3. **Supported Operations**:
   - Aggregations: sum, avg, count, max, min, correlation
   - Filters: Conditional filtering on any column
   - Semantic: Natural language understanding and explanation
   - Lookup: Finding specific records by ID or criteria

4. **Multi-File Intelligence**: When multiple files are uploaded, the system maintains a registry of file summaries and uses LLM-based routing to select the correct dataset for each query.

5. **Context Preservation**: Each response includes the actual data retrieved, allowing you to verify the LLM's interpretation and ensuring transparency.
